      const root = document.querySelector('[data-gallery-dialog-recipe]');
      const dialog = initDialog(root);

      root.querySelectorAll('[data-gallery-title]').forEach((trigger) => {

          root.querySelector('[data-gallery-dialog-title]').textContent =

          const artwork = root.querySelector('[data-gallery-dialog-art]');

          artwork.setAttribute('aria-label', trigger.dataset.galleryTitle);
        });
      });

Axoloth Classes

Structure supplied by the library

  • axo-auto-gridChooses a responsive column count from available width.
  • axo-frameMaintains stable thumbnail and dialog media ratios.
  • axo-card + axo-surfaceProvide neutral item structure.
  • axo-flowMaintains rhythm inside items and dialog content.
  • axo-dialog*Provides the modal layer, panel, regions, and backdrop.
  • axo-buttonProvides neutral dismiss controls.

Customizable Variables

Supported tuning points

  • --axo-item-minMinimum gallery item width.
  • --axo-grid-gapGrid rhythm.
  • --axo-frame-ratioThumbnail and detail media proportions.
  • --axo-dialog-widthMaximum detail panel width.
  • --axo-dialog-paddingPanel inset.
  • --axo-dialog-backdropBackdrop color and opacity.
  • --axo-dialog-durationOpen and close transition timing.

Project-owned CSS + JS

Content remains application data

The image sprite, crop positions, card treatment, typography, and hover styling are local CSS. The small script copies selected content into the shared dialog; Axoloth Behavior still owns open state, ARIA, focus trap, Escape, dismissal, and cleanup.

Mobile + Desktop

Columns respond to available width

Desktop naturally fits several cards. Mobile becomes one column through --axo-item-min. The dialog uses viewport insets and scroll bounds, so content remains reachable without a second mobile-only component.
